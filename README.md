## Overview

SAFE-GEM is a pre-deployment screening framework that audits GenAI-generated explanations for AI-based microseismic event detection **before** they are shown to human expert verifiers.

When an AI model detects a microseismic event in a Distributed Acoustic Sensing (DAS) recording, a Large Language Model (LLM) is often used to generate a natural-language explanation. These explanations can contain hallucinated signal characteristics, overconfident language, or misrepresentations of sensor evidence — properties that could cause engineers to over-rely on incorrect AI decisions.

SAFE-GEM addresses this by:
1. Extracting symbolic signal evidence directly from the DAS recording
2. Generating explanations using five prompting strategies (P1–P5)
3. Auditing each explanation's claims against the computed signal evidence
4. Scoring explanation risk using the **EIRRS** (Explanation-Induced Reliance Risk Score)

> EIRRS is a screening heuristic, not a safety certification score. It identifies explanations that warrant closer human review before deployment.

## Key Findings

| Strategy | EIRRS | UCR | Low-risk % |
|---|---|---|---|
| P1 Raw GenAI | 0.187 | 12.8% | 84.4% |
| P2 Feature-grounded | 0.238 | 1.9% | 74.7% |
| P3 Uncertainty-aware | 0.136 | 5.8% | 97.4% |
| P4 Evidence-constrained | 0.143 | 1.1% | 94.7% |
| **P5 SAFE-GEM** | **0.117** | **1.7%** | **96.6%** |

**Counterintuitive finding:** Feature-grounded prompting (P2) produces the *highest* explanation risk — higher than raw unconstrained GenAI — because technical specificity suppresses hedging behaviour. This finding is stable across six LLMs from five company families.

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/shokor103072/safe-gem.git
cd safe-gem
```

### 2. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 3. Install and start Ollama

Download Ollama from [https://ollama.com](https://ollama.com), then pull the primary model:

```bash
ollama pull qwen2.5:7b
ollama serve
```

For the multi-LLM evaluation, also pull:

```bash
ollama pull llama3.2:latest
ollama pull mistral:7b-instruct
ollama pull gemma2:9b
ollama pull phi3.5:latest
ollama pull deepseek-r1:7b
```

### 4. Download the dataset

Download the Utah FORGE 3-2417 DAS dataset and place the following files in `Dataset/`:

```
Dataset/
├── X_forge.npy      # DAS signal array (494, 1, 361, 2400)
└── y_forge.npy      # Labels (494,)
```

## The EIRRS Metric

EIRRS comprises six theory-grounded components:

| Component | Description | Theoretical basis |
|---|---|---|
| **AER** | Automation Error Risk | Parasuraman & Riley (1997) |
| **EC** | Explanation Confidence | Lee & See (2004) |
| **UTC** | Unsupported Technical Claims | Jacovi & Goldberg (2020) |
| **SA** | Signal Ambiguity | Lee & See (2004) |
| **UO** | Uncertainty Omission | Bhatt et al. (2021) |
| **MI** | Inter-Array Consistency | Endsley (2017) |

Each component is scored 0–3. EIRRS is the normalised weighted average, mapping to three screening tiers:

- **Low** (EIRRS ≤ 0.33): explanation passes pre-deployment screening
- **Medium** (0.33–0.66): explanation flagged for human review
- **High** (> 0.66): explanation blocked, must be regenerated

> These thresholds are provisional research conventions and must be recalibrated for operational deployment.

## The Five Prompting Strategies

| ID | Name | Description |
|---|---|---|
| P1 | Raw GenAI | No constraints — LLM explains freely |
| P2 | Feature-grounded | LLM receives extracted signal features |
| P3 | Uncertainty-aware | LLM required to hedge and mention alternatives |
| P4 | Evidence-constrained | LLM grounded in provided evidence only |
| **P5** | **SAFE-GEM** | **Combines grounding + uncertainty + sensor awareness** |

## Multi-LLM Evaluation

EIRRS rankings were validated across six LLMs:

| ID | Model | Company | Parameters |
|---|---|---|---|
| M1 | Qwen 2.5 | Alibaba | 7B |
| M2 | Llama 3.2 | Meta | 3B |
| M3 | Mistral 7B | Mistral AI | 7B |
| M4 | Gemma 2 | Google | 9B |
| M5 | Phi-3.5 Mini | Microsoft | 3.8B |
| M6 | DeepSeek-R1 | DeepSeek | 7B |

The P5-safest ranking is stable across four of five instruction-tuned models (M1–M5), confirming model-agnosticity in the 3B–9B parameter range. DeepSeek-R1 (M6) requires dedicated vocabulary calibration due to its chain-of-thought architecture.

## Dataset

- **Utah FORGE 3-2417 DAS Dataset** — July 2023 hydraulic stimulation campaign
- 494 labelled windows: 252 microseismic events, 242 noise
- Shape: (494, 1, 361, 2400) — 361 channels, 4.8 s at 500 Hz
- The URDI classifier was trained on Stage 14 Cape EGS Frisco-2-P data and evaluated zero-shot on FORGE, representing a cross-site, cross-domain stress test
